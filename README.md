# ExamEye

AI-powered exam surveillance and investigation platform. Turns passive CCTV footage into intelligent investigation assistants using YOLOv8 for detection and **Gemma 4** (via Ollama or any OpenAI-compatible endpoint) for natural-language reasoning.

## Stack

- Python 3.11+, FastAPI, SQLAlchemy 2.x (SQLite)
- OpenCV for frame extraction
- Ultralytics YOLOv8 for person/object detection
- **Gemma 4** (`gemma4:e2b` by default — fits in ~5 GB RAM at 4-bit quant, runs comfortably on an 8 GB machine)
- Jinja2 templates + a tiny vanilla JS frontend
- WebSockets for real-time alerts

## Quick start (8 GB RAM machine)

```bash
# 1. Install dependencies
uv sync

# 2. Install Ollama (one-time): https://ollama.com/download
#    Then make sure the daemon is running:
ollama serve         # or just launch the macOS app

# 3. Pull Gemma 4 (≈4 GB download, ≈5 GB RAM at runtime)
uv run examye pull-model

# 4. Verify the backend is reachable
uv run examye ping

# 5. Start ExamEye
uv run examye serve
```

Open <http://localhost:8000>.

## Model picker

| Model | Download | Runtime RAM | When to use |
|---|---|---|---|
| `gemma4:e2b` (**default**) | ~4 GB | ~5 GB | 8 GB RAM machines. Multimodal, fast. |
| `gemma4:e4b` | ~6 GB | ~6 GB | If you have 12 GB+ free. Stronger reasoning. |
| `gemma3:1b` | ~0.8 GB | ~1.5 GB | Very tight RAM budgets / Raspberry Pi. |
| `gemma3:4b` | ~3 GB | ~4 GB | If `gemma4:e2b` is unavailable for any reason. |

To switch, set `EXAMYE_LLM_MODEL` (in `.env` or shell) and re-pull:

```bash
EXAMYE_LLM_MODEL=gemma4:e4b uv run examye pull-model
```

## Backend swap (vLLM / OpenAI / stub)

```bash
# Use a vLLM server you started elsewhere
export EXAMYE_LLM_BACKEND=openai
export EXAMYE_LLM_BASE_URL=http://localhost:8001/v1
export EXAMYE_LLM_API_KEY=EMPTY
export EXAMYE_LLM_MODEL=google/gemma-4-e4b-it

# Or — no LLM at all, deterministic templated summaries only
export EXAMYE_LLM_BACKEND=stub
```

If the configured backend isn't reachable at runtime, ExamEye falls back to the stub automatically so the rest of the pipeline keeps working.

## End-to-end flow

1. Admin uploads CCTV footage at `/`.
2. Background pipeline extracts frames (OpenCV), runs YOLOv8 detection, scores suspicion per frame.
3. Medium- and high-severity events push live alerts over WebSocket (`/alerts`).
4. Per-video investigation summary is generated by Gemma 4 (or stub).
5. Admin asks natural-language questions at `/query`; relevant events are retrieved as evidence and answered by Gemma 4 with `[Eid]` citations.

## CLI

```bash
uv run examye serve [--host 0.0.0.0] [--port 8000] [--reload]
uv run examye pull-model [--model gemma4:e2b]
uv run examye models           # list locally cached Ollama models
uv run examye ping             # check whether the LLM backend is reachable
```

## Layout

```
src/examye/
  main.py          FastAPI app wiring
  cli.py           `examye serve | pull-model | models | ping`
  config.py        Pydantic settings (env-prefix EXAMYE_)
  database.py      SQLAlchemy session/engine
  models.py        ORM models (Video, Frame, Event, Summary, Query)
  schemas.py       Pydantic IO schemas
  pipeline.py      Per-video processing orchestrator
  routes/          HTTP + WebSocket endpoints
  services/
    frame_extraction.py   OpenCV frame sampling
    detection.py          YOLOv8 + suspicion heuristics
    summarization.py      Per-video investigation summary
    query.py              NL query (retrieval + answer)
    llm.py                Backend-agnostic OpenAI-compatible client
    ollama.py             Ollama daemon ops: pull / list / ping
    alerts.py             WebSocket pub/sub hub
  templates/       Jinja2 HTML
  static/          CSS + JS
tests/             Pytest suite
```

## Testing

```bash
uv run pytest
```

All tests run offline — the LLM backend is set to an unreachable address inside the test environment so the stub fallback exercises the same code path.
